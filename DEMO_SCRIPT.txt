[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture diagram to see how it works under the hood. (pause)
First, the candidate enters their resume and target role, which is received by the Start node. (pause)
Every input goes directly to our Security Checkpoint node to validate the text, detect prompt injection, and scrub personal details like emails and phone numbers. (pause)
If passed, the request reaches the Interview Orchestrator node, which manages the overall flow. (pause)
The Orchestrator LlmAgent delegates tasks to specialized sub-agents using AgentTools, starting with the Resume Analyzer to parse skills, followed by the Interview Planner to design the difficulty and questions. (pause)
For each interview step, the Orchestrator pauses using a RequestInput node to wait for the human response. (pause)
Each user answer is evaluated by the Feedback Agent for behavioral questions, or the Coding Evaluator for technical coding problems. (pause)
The Coding Evaluator connects to our local MCP Server to run validate_python_syntax checks on the user code. (pause)
Finally, when the interview is complete, the Roadmap Agent analyzes the history and uses MCP tools like get_study_resources and suggest_practice_challenges to generate a comprehensive report. (pause)
